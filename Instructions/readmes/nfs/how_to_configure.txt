Чтобы сконфигурировать NFS-сервер, необходимо поставить некоторые пакеты.
Для каждого дистрибутива эти пакеты свои. Очевидно, что на дистрибутивах
без пакетных менеджеров сервер надо собирать самостоятельно.
Для систем с RPM и Arch-Linux пакет называется nfs-utils.
Для системы с deb-пакетами нужны пакеты nfs-kernel-server, nfs-common.
Для Clear Linux Project (данная система) нужен бандл koji, 
в него входит пакет nfs-utils.


Для установки сервера необходимо настроить конфигурационный файл
exports, после чего перезапустить службу сервера. Для подключения
к серверу необходимо открыть порты 111, 2049.
На системе, где установлен сервер, следует создать отдельного 
пользователя для монтирования NFS на внешней системе.

Для монтирования NFS на UNIX-like системах можно использовать
утилиту mount. 
Пример использования:
sudo mount -t nfs -O uid=1000,iocharset=utf-8 192.168.1.2:/data ~/share
Здесь:
-t nfs => задаёт тип монтируемой файловой системы - nfs;
-O uid=1000,iocharset=utf-8 => задаёт опции для подключения к 
nfs. Опции разделяются запятой, здесь указаны не все опции.
Список общих опций для NFS: https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/5/html/deployment_guide/s1-nfs-client-config-options
Для более подробной информации смотри мануал по mount.
192.168.1.2:/data => [адрес сервера]:[путь до общедоступного каталога];
~/share => путь до каталога на данной системе, куда необходимо 
смонтировать указанный ранее каталог.
Для автоматического монтирования NFS при запуске системы можно использовать
утилиту fstab.
Для автоматического монтирования NFS при наличии доступа к ней
можно использовать утилиту autofs.
